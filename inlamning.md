## Reflektion

Arbetet gick bra och vi följde instruktionerna steg för steg. Vi skapade varsin branch, gjorde flera commits, pushade våra ändringar och lärde oss hur man använder pull requests för att granska varandras arbete.

Samarbetet via GitHub fungerade smidigt, och det var tydligt hur Git hanterar olika versioner av samma fil. Vi uppskattade hur lätt det var att se varandras ändringar och godkänna dem via granskningen.

Det mest utmanande var att förstå hur pull requests fungerar första gången, men efter att vi gjort det en gång blev det tydligt hur flödet hänger ihop.

Ingen merge-konflikt uppstod eftersom vi arbetade i olika delar av textfilen. Vi har dokumenterat detta i filen `konfliktlogg.md`.

Framöver vill vi öva mer på att hantera merge-konflikter och skriva tydligare commit-meddelanden, men i övrigt kändes övningen väldigt lärorik.
